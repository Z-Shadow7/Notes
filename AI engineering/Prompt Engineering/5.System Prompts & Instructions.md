## What is a system prompt?

A system prompt is the **hidden foundation** of every AI app you build. The user never sees it, but it controls everything — how the model behaves, what it refuses, what format it replies in, and what personality it has.

Think of it like a rulebook you give an employee on day one. The user messages are the daily tasks. The system prompt is the employee handbook that shapes *how* those tasks get done.

It is sent at the start of every API call as `"role": "system"` and it applies to **every single response** in that conversation.

---

## The 4 main sections of a system prompt

### 1. Role — who the model is
```
You are a customer support agent for CloudStore, an online file storage company.
You have expert knowledge of all CloudStore products and pricing.
```

### 2. Tone & style — how it talks
```
Always be friendly and concise.
Use plain English — no technical jargon unless the user asks for it.
Never use filler phrases like "Great question!" or "Certainly!".
```

### 3. Rules & constraints — what it must/must not do
This is where you draw hard lines. Very important for production apps.
```
Only answer questions related to CloudStore products.
If someone asks about a competitor, politely redirect to CloudStore features.
Never make up pricing — if unsure, say "I'll check that for you."
Never discuss politics, religion, or anything off-topic.
```

### 4. Output format — how responses are structured
```
Always respond in this format:
- Start with a one-sentence direct answer
- Then give 2-3 bullet points with details if needed
- End with: "Need more help? Type 'human' to reach our team."
```

## Full example — putting it all together

```python
from groq import Groq
client = Groq(api_key="your-groq-api-key")

SYSTEM_PROMPT = """
## Role
You are a customer support agent for CloudStore, an online file storage app.
You know everything about CloudStore plans, features, and billing.

## Tone
Be friendly, warm, and concise.
Use simple everyday English — no tech jargon unless the user asks.
Never start a sentence with "Certainly!" or "Great question!".

## Rules
- Only answer questions about CloudStore.
- If asked about a competitor, say: "I can only speak to what CloudStore offers."
- Never guess at pricing — say "Let me verify that for you" if unsure.
- If the user is angry or upset, stay calm, acknowledge their frustration first.
- Never discuss politics, religion, or anything unrelated to CloudStore.

## Output format
1. One sentence direct answer first.
2. Bullet points with detail if needed (max 3 bullets).
3. Always end with: "Anything else I can help with?"
"""

def support_chat(user_message: str) -> str:
    response = client.chat.completions.create(
        model="llama-3.3-70b-versatile",
        max_tokens=400,
        messages=[
            {"role": "system", "content": SYSTEM_PROMPT},
            {"role": "user",   "content": user_message}
        ]
    )
    return response.choices[0].message.content

# Test it
print(support_chat("How much does the Pro plan cost?"))
print(support_chat("Why is Google Drive better than you?"))
print(support_chat("I've been charged twice this month and I'm furious!"))
```

## Multi-turn conversation — system prompt must persist

The system prompt does NOT automatically carry over between messages. You have to include it in **every API call** manually. Here's the right way to build a chatbot that remembers context:

```python
from groq import Groq
client = Groq(api_key="your-groq-api-key")

SYSTEM_PROMPT = """You are a helpful coding tutor for beginners.
Explain everything simply. Use short examples. Be encouraging."""

# This list grows as the conversation goes on
conversation = []

def chat(user_message: str) -> str:
    # Step 1: add user message to history
    conversation.append({"role": "user", "content": user_message})

    # Step 2: call API with system prompt + full history
    response = client.chat.completions.create(
        model="llama-3.3-70b-versatile",
        max_tokens=400,
        messages=[
            {"role": "system", "content": SYSTEM_PROMPT},  # always first
            *conversation                                    # full history after
        ]
    )

    # Step 3: save assistant reply to history
    reply = response.choices[0].message.content
    conversation.append({"role": "assistant", "content": reply})
    return reply

# Multi-turn conversation — model remembers everything
print(chat("What is a variable?"))
print(chat("Can you give me an example in Python?"))
print(chat("What happens if I don't give it a value?"))
```

## Common mistakes to avoid

| Mistake | Why it's a problem | Fix |
|---|---|---|
| Too vague (`"be helpful"`) | Model has no real direction | Be specific about role + audience |
| No output format defined | Replies are inconsistent in length and style | Always define the structure you want |
| No constraints | Model goes off-topic or makes things up | Add explicit "never do X" rules |
| Forgetting system prompt in multi-turn | Model loses its personality mid-conversation | Always include it in every API call |
| Too many rules (10+) | Model starts ignoring some of them | Keep it to 5–8 clear, focused rules |

---

## Key Insight

> Your system prompt is your product's personality. Two apps using the exact same model can feel completely different just because of what's in their system prompt.

It costs zero extra effort to write a good one. Spend time on it — it is the single highest-leverage thing you write in any AI app.
