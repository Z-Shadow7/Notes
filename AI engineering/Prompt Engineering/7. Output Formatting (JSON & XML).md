## What is it and why does it matter?

By default the model replies in plain conversational text. That's fine for a chatbot — but in real apps you need **structured, predictable output** your code can actually parse and use.

If you're building a product that extracts data from resumes, sends it to a database, and displays it on a dashboard — you can't work with a paragraph of text. You need clean JSON every single time, in exactly the same shape.

## Format 1 — JSON

Most common format in AI engineering. Use it when the model's output needs to go into a database, API, or the frontend of your app.

Tell the model the **exact shape** of the JSON you want — field names, types, what goes in each one.

```python
from groq import Groq
import json

client = Groq(api_key="your-groq-api-key")

response = client.chat.completions.create(
    model="llama-3.3-70b-versatile",
    max_tokens=400,
    messages=[
        {
            "role": "system",
            "content": """You extract information from job postings.
Always respond with valid JSON only. No extra text, no markdown, no explanation.
Use exactly this structure:
{
  "job_title": "string",
  "company": "string",
  "location": "string",
  "salary": "string or null if not mentioned",
  "skills": ["list", "of", "required", "skills"],
  "experience_years": "number or null if not mentioned"
}"""
        },
        {
            "role": "user",
            "content": """
            We're hiring a Senior Backend Engineer at TechCorp in Bangalore.
            You'll need 5+ years with Python, PostgreSQL, and Docker.
            Salary: ₹25-35 LPA.
            """
        }
    ]
)

raw = response.choices[0].message.content
data = json.loads(raw)

print(data["job_title"])   # Senior Backend Engineer
print(data["skills"])      # ['Python', 'PostgreSQL', 'Docker']
print(data["salary"])      # ₹25-35 LPA
```

### Always protect your JSON parsing

Models occasionally add extra text or wrap JSON in markdown backticks. Always use this helper:

```python
import json, re

def parse_json_response(raw: str) -> dict:
    """Safely parse JSON — handles markdown code fences."""
    clean = re.sub(r"```json|```", "", raw).strip()
    try:
        return json.loads(clean)
    except json.JSONDecodeError as e:
        print(f"Failed to parse JSON: {e}")
        return {}
```

## Format 2 — XML tags

Best when you need to extract **multiple separate pieces** from one response and use each one independently. Each tag acts as a named container.

```python
from groq import Groq
import re

client = Groq(api_key="your-groq-api-key")

response = client.chat.completions.create(
    model="llama-3.3-70b-versatile",
    max_tokens=600,
    messages=[
        {
            "role": "system",
            "content": """You are a code reviewer.
Review the code and respond using EXACTLY these XML tags — nothing outside them:

<summary>One sentence overview of what the code does</summary>
<issues>Bullet list of problems found. Write 'None' if no issues.</issues>
<improved_code>The corrected version of the code</improved_code>
<explanation>Plain English explanation of what you changed and why</explanation>"""
        },
        {
            "role": "user",
            "content": """
def get_user(id):
    query = f"SELECT * FROM users WHERE id={id}"
    return db.execute(query)
            """
        }
    ]
)

raw = response.choices[0].message.content

# Extract each section by its tag
def extract_tag(tag: str, text: str) -> str:
    match = re.search(rf"<{tag}>(.*?)</{tag}>", text, re.DOTALL)
    return match.group(1).strip() if match else ""

summary     = extract_tag("summary",       raw)
issues      = extract_tag("issues",        raw)
improved    = extract_tag("improved_code", raw)
explanation = extract_tag("explanation",   raw)

print("SUMMARY:\n",      summary)
print("\nISSUES:\n",     issues)
print("\nFIXED CODE:\n", improved)
print("\nEXPLAINED:\n",  explanation)
```

## Format 3 — Markdown

Use when the output is meant to be **read by a human** — documentation, reports, blog posts. Most chat UIs render markdown automatically.

```python
from groq import Groq
client = Groq(api_key="your-groq-api-key")

response = client.chat.completions.create(
    model="llama-3.3-70b-versatile",
    max_tokens=500,
    messages=[
        {
            "role": "system",
            "content": """You write technical documentation.
Always respond in well-structured Markdown.
Use: ## headings, **bold** for key terms, code blocks for any code,
and a table if comparing multiple things."""
        },
        {
            "role": "user",
            "content": "Write a short doc explaining what an API is, with an example."
        }
    ]
)

print(response.choices[0].message.content)
# Output is clean markdown — ready to display or save as .md
```

## Which format to use when?

| Format | Use it when... |
|---|---|
| **JSON** | Output goes into code — database, API call, frontend display |
| **XML tags** | You need multiple named sections parsed separately |
| **Markdown** | Output is for a human to read — docs, reports, chat |
| **Plain text** | Simple chatbot replies, nothing needs parsing |

---

## Key Insight

> Put the format instruction in the **system prompt**, not the user message — that way it applies to every response automatically.

Always tell the model:
1. What format to use
2. What fields/tags you expect
3. What type each field should be
4. What to do when data is missing (e.g. `null`)

The more specific you are about the shape of the output, the more consistently the model delivers it.
